Reversibility is a Degree

In digital systems, a gate is reversible or it isn't. In analog systems — neural networks, physical dynamics — reversibility becomes a measure. Sho Kuwamoto's thesis asks: can we train a neural network to be as invertible as possible, and what does that cost?

01 · The Problem

What Is Analog Computation?

Digital gates map bits to bits. A NAND gate is irreversible — (1,0) and (0,1) both produce 1. Landauer showed each bit erased costs kT ln 2 energy. Fredkin showed any digital computation can be made reversible by keeping "garbage" outputs. But what about continuous signals?

Digital: Yes or No

NAND gate:
(1,0) → 1
(0,1) → 1
two inputs, one output → irreversible
⊗

Reversibility is binary. A gate either has an inverse or it doesn't. The Fredkin gate is exactly its own inverse.

Analog: How Much?

Neural network:
o = g(Σ wⱼxⱼ)
continuous in, continuous out
many-to-one via saturation
≈

Reversibility is a degree. How invertible is this mapping? How much information leaks into noise? Sho's key insight (1995).

03 · Degrees of Reversibility

Training Invertibility

Sho's insight: don't just build an invertible architecture — train the network to stay invertible. Three penalty terms in the energy function measure how far the network drifts from perfect reversibility.

C₁ — Garbage Statistics

½C₁ Σi∉I [ōᵢ² + (1 − σᵢᵢ)²]
📊

Drives garbage outputs to mean = 0 and variance = 1 across the training set. Without this, garbage values are unpredictable — you'd have no idea what to plug in when running the network backward.

C₂ — Garbage Independence

½C₂ Σi∉I, j≠i σᵢⱼ²
⊥

Penalizes correlation between garbage outputs. If two garbage dims are correlated, they carry redundant information — wasting the capacity you paid for. Each garbage line should encode something independent.

C₃ — Volume Preservation

C₃ Σk |1 − det(M⁽ᵏ⁾)|
≅

The deepest term. Pushes each layer's weight matrix toward unit determinant. When det(M) = 1, the transformation preserves phase-space volume — this is Liouville's theorem expressed as a loss function. The rotation-restricted Fredkin gate satisfies this exactly (M is orthogonal). The unrestricted gate uses C₃ to stay close.

⇌

This is the discrete analog of the Poisson bracket {H, f} generating reversible Hamiltonian flow. C₃ measures how far the network's dynamics are from a symplectic map. Training minimizes this distance — learning to be reversible.

05 · The Algebra of Reversibility

From Dagger to Poisson Bracket

Invertibility has a geometry. In category theory it's exact. In physics it's continuous. Sho's C₃ term sits at the intersection.

†
Dagger (Exact)

In a dagger category, every morphism has a formal reverse. f† ∘ f = id. All or nothing — either the gate has an inverse or it doesn't. This is where digital Fredkin lives.

f† ∘ f = idA
{,}
Poisson Bracket (Infinitesimal)

Measures how two observables fail to commute under Hamiltonian flow. Generates reversible evolution. The continuous algebra of symmetry — when {H,f} = 0, f is conserved.

{f,g} = Σ(∂f/∂q · ∂g/∂p − ∂f/∂p · ∂g/∂q)
Ω
Liouville (Volume)

Hamiltonian flow preserves phase-space volume. dΩ/dt = 0. The continuous analog of bijectivity — no states are created or destroyed. Sho's rotation gate satisfies this exactly.

|det J| = 1 ⟺ dΩ/dt = 0
C₃
Sho's Bridge

The C₃ penalty is Liouville's theorem as a loss function. It measures how far each layer's matrix is from volume-preserving — the degree of reversibility, trainable by gradient descent.

C₃ Σₖ |1 − det(M⁽ᵏ⁾)|
‖·‖
Deviation Measure

For any operator T, ‖T†T − I‖ measures deviation from perfect reversibility. This is Sho's C₃ in operator-norm language. Zero means unitary. Positive means dissipation.

‖MTM − I‖ → 0
kT
Landauer Cost

Every bit of information the network destroys costs at least kT ln 2 energy. The garbage dims exist to avoid paying this cost. Information preserved = energy saved.

ΔQ ≥ kT ln 2 per bit erased
06 · Implementation

PyTorch Reference Code

import torch
import torch.nn as nn

class Coupling(nn.Module):
    def __init__(self, dim):
        super().__init__()
        # f_θ can be any neural network
        self.f = nn.Sequential(
            nn.Linear(dim//2, 32),
            nn.ReLU(),
            nn.Linear(32, dim//2)
        )

    def forward(self, x):
        x1, x2 = x.chunk(2, dim=-1)
        y1 = x1                          # identity pass-through
        y2 = x2 + self.f(x1)           # additive shift
        return torch.cat([y1, y2], dim=-1)

    def inverse(self, y):
        y1, y2 = y.chunk(2, dim=-1)
        x1 = y1                          # recover z1 trivially
        x2 = y2 - self.f(y1)           # undo the shift
        return torch.cat([x1, x2], dim=-1)
class ReversibleIris(nn.Module):
    def __init__(self, input_dim=4, hidden_dim=16, layers=4):
        super().__init__()
        # Lift input to higher-dimensional reversible state
        self.embed = nn.Linear(input_dim, hidden_dim)
        # Stack of bijective coupling layers
        self.couplings = nn.ModuleList(
            [Coupling(hidden_dim) for _ in range(layers)]
        )
        # Classifier reads only the first half of final state
        self.classifier = nn.Linear(hidden_dim//2, 3)

    def forward(self, x):
        z = self.embed(x)           # ℝ⁴ → ℝ¹⁶
        for layer in self.couplings:
            z = layer(z)             # bijective transform
        z_used, z_garbage = z.chunk(2, dim=-1)
        # z_garbage: unused dims, carry invertibility trace
        logits = self.classifier(z_used)
        return logits

    def inverse(self, z_L):
        # Exact inverse through coupling stack
        z = z_L
        for layer in reversed(self.couplings):
            z = layer.inverse(z)
        return z               # recovers (x, g₀)
from sklearn.datasets import load_iris
import torch.optim as optim

# Load and normalise Iris
data = load_iris()
X = torch.tensor(data.data, dtype=torch.float32)
y = torch.tensor(data.target, dtype=torch.long)
X = (X - X.mean(0)) / X.std(0)

model = ReversibleIris(hidden_dim=16, layers=4)
opt   = optim.Adam(model.parameters(), lr=1e-3)

for epoch in range(500):
    logits = model(X)
    loss   = nn.functional.cross_entropy(logits, y)

    opt.zero_grad()
    loss.backward()
    opt.step()

    if epoch % 100 == 0:
        acc = (logits.argmax(1) == y).float().mean()
        print(f"Epoch {epoch}: loss={loss:.4f}, acc={acc:.3f}")
